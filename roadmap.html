Roadmap

What’s coming next for the universal proof layer: SDKs, certificates, API keys, quotas, dashboard, and a light distributed validation layer. Built openly, privacy-first, and edge-native.

Current: v0.1 (Public Beta) Infra: Cloudflare Workers + KV Focus: DX & reliability

v0.2 — Developer Experience

  • JavaScript SDK (browser & Node): createFromText/File/Hash, verify
  • PDF certificate generation (downloadable proof with timestamp & signature)
  • Verify UX: buttons “Copy as cURL” & “Copy JSON”
  • Docs refresh: Why TimeProofs? limits, examples

Goal: integrate in minutes; proofs visible and portable for humans.

v1.0 — Productization

  • API keys + usage quotas (Free/Pro/Team)
  • Dashboard (usage analytics, CSV export, key rotation)
  • Webhooks (success/failure callbacks)
  • Billing (Stripe) — predictable, no “gas”

Goal: reliable operations at scale with predictable pricing.

v2.0 — Validation Layer

  • ProofChain (light distributed validation / anchoring)
  • SDKs: Python & Go
  • Advanced verify (proof bundles, offline verification)

Goal: extra assurance without sacrificing speed or privacy.

Backlog / Ideas

  • Signed agents: per-agent public key & attestations
  • CLI for CI/CD pipelines
  • S3 / GCS integration (hash on upload)
  • Org features: roles, audit export, SIEM
  • Trusted timestamping (RFC 3161 compatibility – research)

Shaped by real-world usage and partner feedback.

Principles

  • Privacy-first: only hashes, never content
  • Open & simple: minimal protocol, verifiable by anyone
  • Edge-native: low latency, global scale
  • Predictable cost: no gas, no chain fees
