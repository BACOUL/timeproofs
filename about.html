What TimeProofs does

1) Hash locally
Compute a SHA-256 of any content: AI output, file, email, webhook, dataset.
2) Timestamp via API
POST the hash to /timestamp. We return an ISO time and an HMAC signature.
3) Verify publicly
Anyone can call /verify?hash=… to confirm existence + timestamp + signature.

Security & privacy

  • Input: hash (64-hex), optional type and small meta
  • Output: ISO timestamp, HMAC-SHA256 signature, verify_url
  • Storage: Cloudflare KV → (hash, timestamp, signature, optional type/meta)
  • Transport: HTTPS; no cookies; minimal edge logs
  • Verification: read-only, idempotent, public
