How it works

Model: Hash locally ‚Üí Signed timestamp ‚Üí Public verification

1) Hash locally

const buf = new TextEncoder().encode(text);
const digest = await crypto.subtle.digest('SHA-256', buf);
const hash = [...new Uint8Array(digest)]
  .map(b => b.toString(16).padStart(2,'0')).join('');

2) Timestamp

const r = await fetch("https://api.timeproofs.io/api/timestamp", {
  method:"POST", headers:{ "Content-Type":"application/json" },
  body: JSON.stringify({ hash, type:"event", meta:{ source:"web" } })
});
const proof = await r.json();

3) Verify anywhere

const res = await fetch(
  "https://api.timeproofs.io/api/verify?hash="+hash
);
const check = await res.json();

FAQ

Do you store my content?

No. We only handle hashes. Original content never leaves your device.

Is there a blockchain involved?

No. Cryptographic timestamps and public verification without blockchain overhead.

How do I verify a proof?

Use our verify endpoint or any ProofSpec-compliant client. The proof is self-describing.

How does this help with the EU AI Act?

It provides timestamped, verifiable trails for AI outputs and model interactions to meet traceability needs.
