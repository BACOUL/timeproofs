Protocol (ProofSpec)

Defines how to create, sign, timestamp, and verify proofs that a specific data artifact existed at a point in time and has not changed since, without exposing the original content.

1. Abstract

ProofSpec binds a content-derived SHA-256 hash to a signed timestamp and a verification method. Transport-agnostic. Verifiable by anyone. Original content stays local.

2. Model & Terms

Actors

  • Author: computes the hash locally.
  • Timestamping Service: signs tuple (hash, ts) and returns a proof.
  • Verifier: checks integrity and signature.

Core flow

  1. Hash input M with SHA-256 ‚Üí hash 64-hex.
  2. POST hash (+ optional meta) ‚Üí get signed Proof.
  3. Verify signature and hash equality. Apply time policy on ts if needed.

5. Algorithms

5.1 Hashing

SHA-256. Text = UTF-8. Lowercase 64-hex.

5.2 Signing

HMAC-SHA256 with service key kid.

message = `${hash}|${ts}`

5.3 Verification

  1. Recompute hash.
  2. Verify sig for message and kid.
  3. Validate spec/alg and any time policy.

Planned: Ed25519 in ProofSpec v1.1.

6. Security & Privacy

  • Hash-only. No content transmitted.
  • No personal data in meta.
  • Key rotation. Bind policy to kid.
  • NTP clock. Drift ‚â§ 5 s.
  • Rate limiting per IP and per key.

10. Reference Implementation

Create (cURL)

curl -sS -X POST https://api.timeproofs.io/api/timestamp \
  -H "Content-Type: application/json" \
  --data '{"hash":"<64-hex>","type":"event","meta":{"source":"spec"}}'

Verify (cURL)

curl -sS "https://api.timeproofs.io/api/verify?hash=<64-hex>"

JS (fetch)

// Create
const resp = await fetch("https://api.timeproofs.io/api/timestamp",{
  method:"POST",
  headers:{ "Content-Type":"application/json","Accept":"application/json" },
  body: JSON.stringify({ hash:"<64-hex>", type:"event", meta:{ source:"spec" } })
});
const proof = await resp.json();

// Verify
const ok = await fetch("https://api.timeproofs.io/api/verify?hash=<64-hex>",{
  headers:{ "Accept":"application/json" }
}).then(r=>r.json());
