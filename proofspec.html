ProofSpec — TimeProofs Open Protocol for Digital Proof of Existence

Defines how to create, sign, timestamp, and verify proofs that a specific data artifact existed at a point in time and has not changed since, without exposing the original content.

1. Abstract

ProofSpec binds a content-derived hash to a signed timestamp and a verification method. Transport-agnostic. Verifiable by anyone. Original content never leaves the device.

2. Model & Terms

Actors

  • Author: computes the hash locally.
  • Timestamping Service: signs a tuple (hash, ts, meta).
  • Verifier: checks integrity and signature.

Core flow

  1. Hash input M with SHA-256 → hash as sha256:<hex64>.
  2. Submit hash + optional meta → receive signed Proof.
  3. Verify signature, time bounds, and hash equality.

3. Data Model

{
  "spec": "proofspec/1.0",
  "hash": "sha256:<64-hex>",
  "alg":  "hmac-sha256",
  "ts":   "2025-10-19T14:00:00Z",
  "meta": { "type":"event", "source":"api", "nonce":"base64url" },
  "sig":  "base64url(hmac(secret, message))",
  "kid":  "key-id-or-issuer",
  "ver":  { "endpoint":"https://.../api/verify", "method":"GET" }
}

Message format below. meta optional. Exclude personal data.

    "meta":{"type":"object","additionalProperties":true},

    "ver":{
      "type":"object",
      "required":["endpoint","method"],
      "properties":{
        "endpoint":{"type":"string","format":"uri"},
        "method":{"type":"string","enum":["GET","POST"]}
      }
    }
  },
  "additionalProperties":false
}

5. Algorithms

5.1 Hashing

Use SHA-256. Text is UTF-8. Emit sha256:<hex64>.

5.2 Signing

HMAC-SHA256 with service key kid. Build UTF-8 message:

message = hash + "\n" + ts + "\n" + base64url(JSON.stringify(meta || {}))

5.3 Verification

  1. Recompute hash from input; compare.
  2. Rebuild message; verify sig using key for kid.
  3. Check spec, alg, and policy on ts.

5.4 Transport

Out of scope. Reference HTTP endpoint in ver.endpoint.

6. Security & Privacy

  • Hash-only: content not transmitted.
  • No personal data in meta.
  • Keys: rotate; bind kid to rotation policy.
  • Replay: include nonce in meta if needed.
  • Time: NTP-synchronized service clocks.

7. Regulatory Mapping

EU AI Act / Data Act / DSA

Supports traceability logs, integrity checks, and auditability for generated/processed content.

GDPR

No personal data required. Keep meta non-identifying.

Complements C2PA: ProofSpec focuses on existence + integrity, not full provenance chains.

8. Versioning

  • spec = proofspec/MAJOR.MINOR.
  • MINOR is backward compatible. MAJOR breaks.
  • Reference implementation tags match protocol.

11. FAQ

Is blockchain required?

No. Standard hashing and signatures. Lower cost. Higher privacy.

Can I verify offline?

Yes, if you have the proof object and the key policy for kid.
