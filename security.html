Security

Coordinated disclosure • Edge-native • Hash-only design

Coordinated Disclosure

Report vulnerabilities privately to security@timeproofs.io. Optionally encrypt using our PGP key published at /pgp.txt. Our policy and contacts are mirrored in /.well-known/security.txt.

  • We will acknowledge receipt, assess impact, and coordinate timelines.
  • Please do not exploit, modify, or access data beyond what’s required to demonstrate the issue.
  • No bug bounty in v0.1. We publish acknowledgments for qualifying reports.

Scope

  • Public API endpoints: /api/timestamp, /api/verify
  • Website static pages and verification UI
  • Edge runtime on Cloudflare Workers + KV

Out of scope

  • Denial-of-service without a novel bypass
  • Self-XSS requiring user pasting code
  • 3rd-party platform issues outside our control

Cryptography

  • Hashing: SHA-256 client-side or by user systems
  • Signing: HMAC-SHA256 over hash + "\n" + timestamp + "\n" + base64url(meta)
  • Transport: TLS only
  • Key management: rotation by kid policy

See ProofSpec for protocol details.

Operational Hardening

  • Minimal data design: hashes + small optional metadata only
  • Strict input validation and size limits
  • Rate limiting and abuse detection at the edge
  • Clock sync and signed timestamps
