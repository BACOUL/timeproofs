AI output provenance

Problem. Users and auditors need proof that an output existed at time T and has not changed.

Solution. Hash the output locally. Create a timestamp proof. Share the public verify URL.

// helper: string ‚Üí sha256 (browser)
async function sha256Hex(str){
  const buf = new TextEncoder().encode(str);
  const digest = await crypto.subtle.digest('SHA-256', buf);
  return [...new Uint8Array(digest)].map(b=>b.toString(16).padStart(2,'0')).join('');
}
const API="https://api.timeproofs.io/api";
const hash=await sha256Hex(output);
const proof=await fetch(`${API}/timestamp`,{
  method:"POST",
  headers:{ "Authorization":"Bearer <TOKEN>","Content-Type":"application/json" },
  body:JSON.stringify({ hash, meta:{ type:"gen", model:"gpt", app:"agentX" } })
}).then(r=>r.json());

Endpoint. POST /api/timestamp ‚Ä¢ Hash-only. Avoid PII in meta.

Contracts, emails, files

Problem. Prove existence and integrity without sharing private content.

Solution. Hash the file locally. Timestamp the hash. Present the proof on demand.
