‚öôÔ∏è Quickstart

  1. Compute a SHA-256 hash locally (text/file/output).
  2. POST /timestamp with the hash (+ optional type & meta).
  3. Receive { ok, hash, timestamp, signature, verify_url, type?, meta? }.
  4. Anyone can GET /verify?hash=... later.

üîê Local hashing (browser)

const buf = new TextEncoder().encode(text);
const digest = await crypto.subtle.digest('SHA-256', buf);
const hash = [...new Uint8Array(digest)].map(b => b.toString(16).padStart(2,'0')).join('');

Then POST the hex hash to /timestamp.

üßæ Verify server signature locally

We sign message = `${hash}|${timestamp}` with HMAC-SHA256 using a private secret. Verify on your side:

Node.js

import crypto from "node:crypto";
function verify({ hash, timestamp, signature }, secret){
  const msg = `${hash}|${timestamp}`;
  const sig = crypto.createHmac("sha256", secret).update(msg).digest("hex");
  return crypto.timingSafeEqual(Buffer.from(sig,'hex'), Buffer.from(signature,'hex'));
}

Python

import hmac, hashlib
def verify(payload, secret: bytes) -> bool:
    msg = f"{payload['hash']}|{payload['timestamp']}".encode()
    sig = hmac.new(secret, msg, hashlib.sha256).hexdigest()
    return hmac.compare_digest(sig, payload["signature"])

The server secret is never exposed. You only verify consistency with what the server claims.

üìè Limits & best practices

  • Hash format: 64-hex lowercase SHA-256.
  • type ‚â§ 32 chars. meta JSON ‚â§ 1 KB. No personal data.
  • Idempotent: same hash returns the same proof payload.
  • CORS: GET /verify and POST /timestamp allowed from browsers.

üîí Security notes

  • No content uploads. Only hashes are processed.
  • Transport: HTTPS only.
  • Signature: HMAC-SHA256 over ${hash}|${timestamp}.
