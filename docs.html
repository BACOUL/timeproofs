Quickstart — Browser

Compute SHA-256 locally, then call the API.

Minimal flow

{
  "flow": "sha256(input) → POST /timestamp → { id, hash, timestamp, signature, verify_url }",
  "verify": "GET /verify?hash=..."
}

Fetch example

// hash = your 64-hex SHA-256
const res = await fetch('https://timeproofs-api.jeason-bacoul.workers.dev/api/timestamp', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ hash, type: 'event', meta: {} })
});
const proof = await res.json();

Quickstart — Node.js

Create and verify

import crypto from 'node:crypto';

const hash = crypto.createHash('sha256').update('hello').digest('hex');

const r = await fetch('https://timeproofs-api.jeason-bacoul.workers.dev/api/timestamp', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ hash, type: 'event' })
});
const proof = await r.json();

const v = await fetch('https://timeproofs-api.jeason-bacoul.workers.dev/api/verify?hash=' + proof.hash);
const check = await v.json();

Endpoints

POST /timestamp

Create a proof of existence for a SHA-256 hash.

POST https://timeproofs-api.jeason-bacoul.workers.dev/api/timestamp
Content-Type: application/json

{ "hash": "64hexsha256", "type": "event", "meta": {} }

Returns

{
  "ok": true,
  "id": "hashprefix:sigprefix",
  "hash": "64hexsha256",
  "timestamp": "2025-01-01T00:00:00.000Z",
  "signature": "hex_hmac256",
  "verify_url": "https://timeproofs-api.jeason-bacoul.workers.dev/api/verify?hash=..."
}

GET /verify

Check if a hash exists and validate its signature.

GET https://timeproofs-api.jeason-bacoul.workers.dev/api/verify?hash=64hexsha256

Returns

{
  "ok": true,
  "found": true,
  "hash": "64hexsha256",
  "timestamp": "2025-01-01T00:00:00.000Z",
  "type": "event",
  "meta": {},
  "signature": "hex_hmac256"
}

Hashing Guidance

  • Text: UTF-8; normalize line endings to \n; optional trim().
  • JSON: canonicalize (sort keys, remove extra spaces) before hashing.
  • Files: hash the raw binary locally (never upload files to the API).
  • Expected format: 64-hex SHA-256 (^[a-f0-9]{64}$).

Limits & Fair Use

  • Payload ≤ ~1 KB (hash + tiny meta).
  • Public beta: no auth required; reasonable rate limiting may apply.
  • Storage: distributed KV (hash, timestamp, signature, optional type/meta).

Security Model

  • Integrity: server computes HMAC-SHA256 over hash + timestamp with a private secret.
  • Privacy: only hashes are stored; your original content never leaves your device.
  • Verification: the returned signature and timestamp are checked on read.
