⚙️ Quickstart

1) cURL — ping

curl -s https://api.timeproofs.io/api/ping

2) cURL — create a proof

curl -s -X POST https://api.timeproofs.io/api/timestamp \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"hash":"<64-hex-sha256>","meta":{"source":"docs"}}'

3) cURL — verify a proof (GET)

curl -s "https://api.timeproofs.io/api/verify?hash=<64-hex-sha256>"

4) cURL — verify a proof (POST)

curl -s -X POST https://api.timeproofs.io/api/verify \
  -H "Content-Type: application/json" \
  -d '{"hash":"<64-hex-sha256>"}'

5) JavaScript (fetch)

const res = await fetch("https://api.timeproofs.io/api/timestamp",{
  method:"POST",
  headers:{ "Authorization":"Bearer <TOKEN>","Content-Type":"application/json" },
  body: JSON.stringify({ hash:"<64-hex-sha256>", meta:{ source:"web" } })
});
const proof = await res.json();

6) Node.js (verify HMAC)

import crypto from "node:crypto";
function verify({ hash, ts, sig }, secret){
  const msg = `${hash}|${ts}`;
  const expect = crypto.createHmac("sha256", secret).update(msg).digest("hex");
  return crypto.timingSafeEqual(Buffer.from(expect,'hex'), Buffer.from(sig,'hex'));
}

🔐 Client-side hash (browser)

const buf = new TextEncoder().encode(text);
const digest = await crypto.subtle.digest('SHA-256', buf);
const hash = [...new Uint8Array(digest)].map(b => b.toString(16).padStart(2,'0')).join('');

Send only the 64-char hex hash.

🧾 Verify a proof

Use GET /verify?hash=… or POST /verify to validate a proof (hash, timestamp, signature).

Signature format: HMAC-SHA256(secret, `${hash}|${ts}`). Server clock: UTC with ±5 s tolerance.

🔒 Security & Best Practices

  • Hash on client. Never upload original content.
  • Keep meta minimal. Avoid PII.
  • Signatures: alg="HMAC-SHA256", rotating secrets with kid.
  • Planned (v0.2): /.well-known/jwks.json for public key discovery.
  • TLS 1.3. CORS allowed origin: https://timeproofs.io.
  • Idempotency: POST /timestamp is idempotent for the same hash.
